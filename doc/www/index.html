Scilint

The Scilab source analyzer

Scilint is a tool to analyze Scilab code, developed within the Richelieu project.

Download version 0.1

Scilint checks Scilab code files to find possible problems.

Local Analyzes

Variable not initialized

Although it is possible in Scilab to access a variable in an upper scope, it is not a good practice (such variables should be passed as arguments), since it is also often an error (a mispelled variable, for example). Also, it can prevent optimisation by a JIT compiler in the future.

Unused arguments

An argument that appears as needed by a function in its arguments and is not used is usually a good sign of a mistake, somewhere in the function.

Documentation

Installation

In order to use this version of Scilint, you need at least OCaml 4.00.

On Linux or OSX, we recommend the use of the OPAM package manager:

opam switch 4.00.1

Once you have OCaml 4.00.1 installed :

make

Further informations will come on how to install Scilint on Microsoft Windows.

Usage

To analyze your Scilab source code, call Scilint with: 

./scilint file.sci
Scilint will parse the file and then analyze it looking for some useful informations to give. Those informations will be printed with location information, so you can easily find your way to the right code.
