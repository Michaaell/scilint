Scilint checks Scilab code files to find possible problems.

Function Analyzes

Variable not initialized

We think that it's important to point out when an 'not-initiliazed' variable is used in the body of a function for severals reasons. The first one is because it can lead to errors or unexpected beahvior. The second one is because it prevents some optimizations

Unused arguments

We feel like this is important because it allow some optimizations. Also it may be an indication that the developer made a mistake.

Documentation

Installation

In order to use Scilint you need OCaml 4.00.

On Linux or OSX, we recommend you use the OPAM package manager:

opam switch 4.00.0

Once you have OCaml 4.00 installed :

make

Further informations will come to install Scilint on Microsoft Windows.

Usage

Scilint will deliver some warnings if it encouters some cases we described.

To analyze your Scilab source code, you can just call Scilint : 

./scilint file.sci
Scilint will parse your file and then analyze it looking for some useful informations to give. Those informations will be deliver with location indication so you can easily find your way to the right code.
